01 / DIAGNOSIS

What's not working today.

Before proposing anything, here's how I'm reading the current state of the app — the patterns that hold up, and the ones that need to change. This is the brief I'm designing against.

Inconsistency

Each module speaks a different dialect

Pomodoro panel feels like a control room; Tasks board is empty cards; Habits is data-dense rows; Accomplishments is one big black box. The vocabulary should be shared — type scale, panel grammar, button language, spacing rhythm.

Hierarchy

Everything sits at the same value

Most surfaces are #0F0F12-ish on #0A0A0C-ish. Cards melt into the background. The eye has nowhere to land. We need at least three depth levels and confident contrast between them.

Tan mode

Reads as bleached, not warm

Light surfaces are too uniformly tan with low-contrast text. The mode should be a deliberate "daylight" companion to dark — paper-like, with proper ink, not a tinted dark theme.

Mobile

Desktop layouts squeezed onto a phone

Three-column boards, side panels, and sidebars don't survive 375 px. Mobile needs its own grammar — a bottom tab bar, a single-column rhythm, and one clear primary action per screen.

Tasks

Three empty caverns

The Burner / Active / Completed lanes give equal real-estate to states with very different volumes. Active deserves the most weight. Cards need a card system, not column-and-pray.

Pomodoro / Timer

Controls float, settings sprawl

Play / reset / skip drift below the dial without grouping. The right rail mixes session config, presets, labels and stats with no hierarchy. Group them, scale them, give the dial more room.

Accomplishments

The PDF preview is trapped

A document inside a card inside a sidebar inside a page is too many containers. Files want a dedicated reading surface — either a split view or a focus mode that lets the document breathe.

Theme panel

It's a list, not a preview

Background, filter, motion are picked blind. The panel should let you see the change before you commit — a live canvas you're previewing into, not a stack of radio buttons.

Voice

Space metaphor under-used

The cover is gorgeous. Inside, the metaphor goes mute. The orbit / cabin / log language can carry through to copy, micro-states, empty states, and labels without becoming kitsch.

02 / DIRECTIONS

Three ways forward.

Each direction commits to a different center of gravity. They share the eight aspects you asked about — concept, sidebar, layout, timer, tasks, mobile, themes, and what changes — but resolve them in distinctly different vocabularies.

01
Direction · A

Obsidian Observatory

Cinematic and quiet. The whole app feels like a single observation deck looking out into space. One ambient backdrop persists across modules; UI floats over it as glass and matte panels with strong typographic hierarchy. A single restrained red accent does all the highlighting work.

Solves · 01

The PDF lives in its own breathable surface — full vertical, with explicit page chrome (page count, size, focus button). It's not nested inside a card.

10 / NEXT

What I'd want to decide next.

A short list of follow-up calls before any code touches keys. None of this is implementation — it's the next round of design conversation.

01

Tan-mode accent strength

Should the Mars red darken in tan mode for AA contrast, or stay identical for brand consistency? I lean toward darken.

02

Sidebar vs top nav

The recommendation keeps the icon rail. If you'd prefer Direction C's top nav (more editorial), we can swap — it's the same system underneath.

03

Calendar & Habits detail

I sketched the system; full mocks for Calendar and Habits (today / manage / review) are the next pass.

04

Empty states

Each module needs a hand-crafted empty state (Tasks, Log, Habits, Calendar). Space-themed copy, real illustration TBD.

05

Onboarding

The current "↓ scroll or arrow keys — enter to open" cover is great. We should design how it transitions into the cabin once selected.

06

Sound & ambient motion

If we're adding sound and orbit motion, we should design the off-state and the on-state with care so they feel additive, never noisy.
